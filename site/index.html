Keep your world-model
honest with itself.

You hold a set of natural-language claims. This tool does not parse them — you paste its prompt into whatever LLM you like; the LLM returns a typed logical IR as JSON (models are reliable at structured JSON, unreliable at solver syntax); you paste that back. You never read logic syntax — only your sentence and our plain-English rendering of what it became. A trusted, test-pinned compiler turns the IR into SMT for Z3 (wasm, entirely in this tab), which returns the smallest set of your claims that can't all be true, the least-entrenchment way out, and what you never asserted but are now committed to. The round-trip is the seam where language meets logic — surfaced on purpose, legible, never hidden. Goal: consistency, not truth.

Step 1 — formalize with your LLM

Claim → prompt

Type one or more natural-language claims. Build the prompt, copy it, paste it into your LLM (it already contains your current registry so the LLM reuses your symbols).

Step 2 — paste the IR back

IR JSON → trajectory

Paste exactly the JSON your LLM returned. It is the typed IR, not solver syntax. Malformed IR or type errors are surfaced — never silently guessed. Each claim then shows as plain English you confirm.

What you're committed to

When the set is consistent, Z3 is asked, for every boolean atom, whether the whole set entails it — the things you never asserted but cannot now deny. Shown automatically below the trajectory. (Free-form entailment queries over the full IR are on the roadmap.)

What this is — and what it is not

The seam is the point. Three places this is honest about, none hidden:
  1. NL → IR faithfulness is done by you + your LLM, in the open. You confirm the plain-English render of what each sentence became — never logic syntax. The tool guarantees consistency of the formalization, never that it faithfully captures your sentence. That stays yours.
  2. Z3 can answer unknown on hard quantified fragments. When it does, it says so. It is never silently treated as "consistent".
  3. Vocabulary discipline (reuse a symbol, don't mint a synonym) — the prompt carries your current vocabulary so the LLM reuses it; mismatches surface as a type error from Z3, not a silent guess.
The goal is consistency, not truth: the tool never claims your claims are right — only whether they can all hold at once (the smallest, and only fully-mechanizable, part), the least-entrenchment way out, and what they jointly force.

Engine: a Rust core (wmt-core, wasm) owns the typed IR, the IR↔SMT compiler, the English renderer, the trajectory, and the analysis driver — it does not solve. The solver is Z3 4.16 (wasm), in this tab. The IR→SMT compiler, the minimal-conflict shrink, and the min-entrenchment optimal repair are verified end-to-end against the native Z3 binary (8 tests, incl. a quantified contradiction and an arithmetic one — not toy propositional). First load fetches ~34 MB of Z3 wasm (then cached).
